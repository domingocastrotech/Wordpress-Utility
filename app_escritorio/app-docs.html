00

Pantalla de inicio — Modo Docker

🔌

Al abrir la aplicación, antes de cargar la interfaz principal, aparece un diálogo de selección de modo. Esto determina contra qué daemon Docker se ejecutarán todas las operaciones.

1 Modo local
Conecta con el Docker Desktop instalado en el mismo equipo. Sin configuración adicional. Si Docker Desktop no está activo al elegir este modo, la app intentará arrancarlo automáticamente y esperará hasta que responda.
2 Modo remoto
Conecta a un Docker daemon en otro equipo. Al seleccionarlo aparecen dos opciones:

• Lista LAN: hosts detectados automáticamente en la red local mediante ARP.
• Manual: introduce una IP, dominio o URL completa. Formatos aceptados: 192.168.1.50, 192.168.1.50:2375, mi-dominio.com, tcp://host:puerto, ssh://usuario@host.
⚠️ Requisito modo remoto: El equipo destino debe tener el daemon Docker accesible por TCP/SSH (puerto 2375 sin TLS o 2376 con TLS). El firewall del equipo destino debe permitir la conexión entrante.
💡 Cancelar: Si cierras el diálogo sin seleccionar modo, la aplicación se cierra limpiamente. Puedes cambiar de modo en cualquier momento desde el botón Cambiar modo Docker en el panel principal.

02

Pestaña Contenedores

📦
Contenedores Lista en tiempo real de todos los contenedores Docker

Muestra una tabla con todos los contenedores (arrancados y parados) del daemon activo. Se actualiza automáticamente cada 30 segundos. La selección se mantiene tras cada refresco.

Cada fila muestra: nombre, estado (con icono descriptivo: iniciando, corriendo, parado, etc.), y puertos mapeados.

Refrescar Fuerza una actualización inmediata del listado de contenedores.
Arrancar Arranca los contenedores seleccionados. Operación no bloqueante: la app sigue respondiendo y muestra un spinner mientras la acción se ejecuta en segundo plano.
Apagar Para los contenedores seleccionados. También funciona en segundo plano.
Arrancar todo Arranca todos los contenedores existentes de una sola vez.
Apagar todo Para todos los contenedores en ejecución.
💡 Selección múltiple: Puedes seleccionar varios contenedores con Ctrl+clic o Shift+clic para operar sobre todos a la vez con Arrancar / Apagar.
03

Pestaña Volumes

💾
Volumes Gestión de volúmenes Docker en modo local y remoto

Muestra todos los volumes del daemon activo con nombre, driver, scope, uso y mountpoint.

Al seleccionar un volume, la columna derecha muestra qué contenedores lo están usando. Si no hay ninguno, aparece como no usado.

Refrescar volumes Recarga el inventario de volumes y su uso real por contenedor.
Crear volume Crea un volume con el driver elegido (por defecto local).
Inspeccionar Muestra el JSON de docker volume inspect para uno o varios volumes.
Clonar volume Crea un volume destino y copia todo el contenido desde el origen.
Vaciar volume Elimina todo el contenido de un volume sin borrar el volume en sí.
Eliminar volume Borra uno o varios volumes seleccionados (forzado) con confirmación previa.
Prune volumes Limpia volumes no usados. La app intenta docker volume prune --all -f y, si el daemon no soporta --all, hace fallback automático a docker volume prune -f.
⚠️ Nota importante sobre prune: un volume no se elimina si está referenciado por cualquier contenedor (incluso detenido). Si nunca estuvo asociado y no hay referencias, sí puede eliminarse en prune.
04

Pestaña Perfiles

🗂️
Perfiles Grupos de contenedores para operar en lote

Un perfil es un conjunto de contenedores con nombre personalizado. Permite arrancar o parar varios contenedores relacionados (por ejemplo WordPress + MariaDB + phpMyAdmin) en un solo clic.

La app soporta dos almacenes: privado (por usuario local en %LOCALAPPDATA%\WordPressUtilidades\private_profiles.json) y remoto (compartido en volumen Docker wpu_profiles_remote en /data/profiles.json).

Guardar / Actualizar Guarda el perfil con el nombre indicado y los contenedores marcados en la lista. Si ya existe, lo actualiza.
Eliminar Elimina el perfil seleccionado tras confirmación.
Quitar del perfil Quita del perfil actual los contenedores que selecciones en la lista de la derecha.
Copiar a remoto / privado Copia el perfil seleccionado al otro almacén. Si ya existe en destino, pide confirmación para sobrescribir.
Arrancar perfil Arranca todos los contenedores del perfil seleccionado en segundo plano.
Apagar perfil Para todos los contenedores del perfil seleccionado.
Refrescar perfiles Recarga la lista de perfiles desde el archivo .ini.
Limpiar selección Vacía el editor de perfil sin guardar cambios.
💡 Marcado visual: Al seleccionar un perfil, los contenedores que ya forman parte de él se muestran con la etiqueta (En el perfil seleccionado) para diferenciarlos al editar.
05

Pestaña Networks

🌐
Networks Gestión completa de redes Docker

Lista todas las redes Docker del daemon activo con su nombre, driver (bridge, host, overlay…) y número de contenedores conectados.

Al seleccionar una red, el panel inferior muestra los contenedores ya conectados (lista informativa) y la lista de contenedores disponibles para conectar o desconectar.

Crear network Solicita nombre y driver— crea una nueva red Docker vacía.
Renombrar network Crea una red nueva con el nombre indicado, migra todos los contenedores y elimina la antigua.
Eliminar network Elimina la red seleccionada tras confirmación. Solo funciona si no tiene contenedores conectados.
Conectar Conecta uno o varios contenedores seleccionados de la lista de objetivos a la red activa. Soporta selección múltiple.
Desconectar Desconecta uno o varios contenedores seleccionados de la red activa.
06

Pestaña Historial

📋
Historial Registro de todas las operaciones realizadas

Muestra el contenido del archivo historial_gestor.log. Cada línea tiene el formato [fecha hora] [NIVEL] Acción | Objetivo | Detalle.

El scroll horizontal y vertical se mantiene en su posición al refrescar, para no perder el hilo de lectura.

Filtro por nivel Desplegable para ver solo líneas OK, ERROR, INFO, o TODOS.
Búsqueda de texto Campo de texto para filtrar líneas que contengan el término buscado (sin distinción de mayúsculas ni acentos).
Refrescar Recarga el archivo de historial manteniendo los filtros activos y la posición de scroll.
Limpiar filtro Resetea el nivel a TODOS y vacía el campo de búsqueda.
Copiar visible Copia al portapapeles el texto actualmente visible (líneas filtradas).
Formato de línea en historial_gestor.log
[DD/MM/AAAA HH:MM:SS] [NIVEL] Acción | Objetivo | Detalle

[16/03/2026 09:45:12] [OK]    START      | wordpress1 | Contenedor arrancado
[16/03/2026 09:46:01] [ERROR] EXPORT     | wordpress1 | Fallo al generar export.sql
[16/03/2026 09:47:33] [INFO]  DOCKER     | desktop    | Docker Desktop iniciado automaticamente
07

Pestaña Logs

🔍
Logs Visor de logs de contenedores Docker

Equivalente a docker logs dentro de la app. Permite ver un número configurable de líneas desde el final, con modos estático y en seguimiento (-f).

Contenedor Desplegable para elegir el contenedor del que visualizar los logs.
Nº líneas Número de líneas desde el final a mostrar (equivalente a --tail N). Por defecto 100.
Ver logs Carga las últimas N líneas. El scroll vertical va al final, el horizontal al inicio (izquierda).
Auto-refresco Checkbox que recarga los logs cada 4 segundos. La posición de scroll x e y se preserva en cada ciclo.
Seguir (-f) Modo streaming en vivo (docker logs -f). Incompatible con Auto-refresco; desactivar uno activa el otro.
Exportar txt Guarda el contenido visible del visor en un archivo .txt a elección del usuario.
Copiar visible Copia al portapapeles el texto actualmente mostrado en el visor.
💡 Comportamiento del scroll: Al pulsar Ver logs la vista salta al final verticalmente pero comienza por la izquierda horizontalmente. Con Auto-refresco activo, ambas posiciones (x e y) se conservan para no interrumpir la lectura.

Gestión avanzada de contenedores

🛠️

Se abre desde la barra lateral con ▶ Gestionar contenedores. Ofrece operaciones de mantenimiento que van más allá del arranque/parada cotidiano.

Gestión avanzada Tabla de todos los contenedores con nombre, estado, imagen y puertos

Muestra nombre, estado, imagen Docker y puertos mapeados de cada contenedor. Permite seleccionar uno y ejecutar acciones sobre él.

Configurar daemon remoto y perfiles compartidos

🌐

Si vas a trabajar en modo remoto, debes preparar dos cosas: exponer el daemon Docker del servidor y crear el volumen compartido donde la app guarda perfiles remotos.

1Exponer daemon Docker en el equipo remoto
Habilita acceso remoto por TCP (2375/2376) o usa SSH. Debes permitir el puerto en firewall para que el cliente pueda conectar.
2Probar conectividad desde el cliente
Comprueba con Test-NetConnection que el host remoto responde antes de abrir la app.
3Crear volumen remoto de perfiles
Crea el volumen wpu_profiles_remote en el daemon remoto donde quedará /data/profiles.json.
4Inicializar y validar profiles.json
Escribe el JSON base y verifica su contenido antes de usar el almacén remoto en la pestaña Perfiles.
Windows remoto (Docker Desktop) - exponer 2375/2376 a LAN
netsh interface portproxy add v4tov4 listenaddress=0.0.0.0 listenport=2375 connectaddress=127.0.0.1 connectport=2375
netsh interface portproxy add v4tov4 listenaddress=0.0.0.0 listenport=2376 connectaddress=127.0.0.1 connectport=2375
New-NetFirewallRule -DisplayName "Docker API 2375" -Direction Inbound -Action Allow -Protocol TCP -LocalPort 2375
New-NetFirewallRule -DisplayName "Docker API 2376" -Direction Inbound -Action Allow -Protocol TCP -LocalPort 2376
Linux remoto (Docker Engine) - daemon.json + firewall
# /etc/docker/daemon.json
{
  "hosts": [
    "unix:///var/run/docker.sock",
    "tcp://0.0.0.0:2375",
    "tcp://[::]:2375",
    "tcp://0.0.0.0:2376",
    "tcp://[::]:2376"
  ]
}

sudo systemctl restart docker
sudo ufw allow 2375/tcp
sudo ufw allow 2376/tcp
Cliente PowerShell - crear e inicializar volumen remoto de perfiles
docker volume create wpu_profiles_remote

$json = '{"version":1,"updated_at":"","updated_by":"","profiles":{}}'
$tmp = Join-Path $env:TEMP 'profiles.json'
Set-Content -Path $tmp -Value $json -Encoding UTF8
Get-Content $tmp -Raw | docker run --rm -i -v wpu_profiles_remote:/data alpine sh -c "cat > /data/profiles.json"

docker run --rm -v wpu_profiles_remote:/data alpine cat /data/profiles.json
docker run --rm -v wpu_profiles_remote:/data alpine cat /data/profiles.json | ConvertFrom-Json
⚠️ Seguridad: El puerto 2375 no usa TLS. Empléalo solo en redes de confianza. Para entornos sensibles, usa ssh://usuario@host o 2376 con TLS real.
✅ En la app: entra en pestaña Perfiles, cambia Almacen a remoto y ya podrás crear/editar perfiles compartidos para todos los clientes conectados al mismo daemon.

Cambiar modo Docker en caliente

🔌

No es necesario reiniciar la app para cambiar de daemon Docker. El botón Cambiar modo Docker en el panel principal abre el mismo selector de conexión visto al iniciar, pero con el modo actual preseleccionado.

1Abrir el selector
Pulsa Cambiar modo Docker en la cabecera. Aparece el diálogo con el modo actual ya marcado y el host anterior en el campo manual si había uno.
2Seleccionar o modificar
Cambia a local, elige otro host LAN de la lista o escribe un nuevo host manual. Pulsa Continuar.
3Refresco automático
Si el modo o el host han cambiado, la app refrescará automáticamente contenedores, redes y perfiles del nuevo daemon. El evento queda registrado en el historial.
💡 Sin cambios: Si confirmas con el mismo modo y host que antes, no se produce ningún refresco ni registro en historial.
Formatos de host remoto aceptados
192.168.1.50            → tcp://192.168.1.50:2375
192.168.1.50:2375       → tcp://192.168.1.50:2375
servidor.local          → tcp://servidor.local:2375
tcp://host:2376         → tcp://host:2376  (sin cambios)
ssh://user@192.168.1.50 → ssh://user@192.168.1.50
