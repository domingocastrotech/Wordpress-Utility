Docker WordPress
Local Environment

Scripts de automatización para desplegar, parar, importar y exportar WordPress en contenedores Docker de forma local, con un menu central para ejecutarlos.

🔧

docker_mariadb_wordpress.bat

Crea y configura el entorno Docker completo desde cero con MariaDB y WordPress.

▶️

gestor_contenedores.bat

Gestion diaria de contenedores: arranque/parada por numero, perfiles, networks, logs e historial.

📦

exportar-wordpress-v3.bat

Exporta la base de datos y los archivos wp-content para subir al hosting.

📥

importar-wordpress.bat

Importa un backup existente al entorno Docker local con navegador de carpetas interactivo.

🧭

menu-utilidades.bat

Menu principal para abrir el resto de scripts y volver automaticamente al menu al finalizar cada uno.

01

Crear entorno desde cero

🔧
📄 docker_mariadb_wordpress.bat

Script de instalación completa con configuracion interactiva. Permite definir nombres, puertos, credenciales, IP de red y dominio de produccion antes de recrear el entorno.

⚠️ Atención: Este script destruye los contenedores, volumenes y red que indiques en la configuracion (por defecto wordpress1, mariadb1, phpmyadmin1, wpdata1, mariadbdata1, wp-network1). Usalo solo para instalaciones limpias.
1 Limpieza del entorno anterior
Para los contenedores activos, los elimina junto con sus volumenes y la red definida. Garantiza una instalacion sin residuos.
2 Creación de red y volúmenes
Crea la red privada y los volumenes configurados para persistencia, validando que los puertos del host no esten ocupados antes de continuar.
3 Arranque de MariaDB
Lanza el contenedor MariaDB usando bitnami/mariadb:latest con la base de datos y credenciales definidas en la configuracion, y espera hasta que responda consultas SQL.
4 Preconfiguración de permisos
Ejecuta un contenedor temporal como root para aplicar chmod 777 y chown 1001:1001 en /bitnami/wordpress, evitando errores de permisos al montar el volumen.
5 Arranque de WordPress
Lanza el contenedor WordPress usando bitnami/wordpress:latest apuntando a MariaDB. Expone puertos configurables (por defecto 8182 HTTP y 8445 HTTPS), y espera hasta que el sitio responda por HTTP.
6 Configurar URLs y reiniciar WordPress
Ajusta URLs con wp search-replace entre dominio de produccion, localhost e IP de red local, y espera a que WordPress vuelva tras el reinicio.
7 Arrancar phpMyAdmin
Crea y arranca un contenedor phpmyadmin:latest conectado a la misma red para administrar MariaDB desde navegador.
✅ Resultado: WordPress + MariaDB + phpMyAdmin listos con valores configurables; por defecto WordPress en http://localhost:8182 y phpMyAdmin en http://localhost:8181.
Fragmento clave — preconfiguración de permisos
:: Evita errores de permisos al montar el volumen
docker run --rm -u root \
  -v wpdata1:/bitnami/wordpress \
  --entrypoint sh bitnami/wordpress:latest \
  -c "chmod -R 777 /bitnami/wordpress && chown -R 1001:1001 /bitnami/wordpress"
02

Gestionar contenedores existentes

▶️
📄 gestor_contenedores.bat

Script de uso diario para arrancar, apagar, agrupar por perfiles, gestionar networks y revisar logs/historial sin recrear contenedores.

1 Comprobar Docker y arrancarlo si hace falta
Verifica que exista docker. Si Docker no esta operativo, intenta abrir Docker Desktop.exe y espera hasta que docker info responda.
2 Listar todos los contenedores
Muestra cada contenedor con su estado actual, salud y puerto principal cuando esta arrancado.
3 Operar por menu (N/P/T/A)
Permite arrancar o apagar por numero, en lote (T/A) y por perfil (F/O). Tambien puedes escribir el nombre exacto de un perfil para accion rapida.
4 Funciones avanzadas del menu
Incluye gestor de perfiles (G), gestor de networks (W: listar, conectar, desconectar, renombrar, crear y eliminar), logs por contenedor (L) e historial (H).
💡 Nota: En la opcion de logs, ahora primero se muestra el listado numerado de contenedores antes de pedir cual elegir.

Exportar al hosting

📦
📄 exportar-wordpress-v3.bat

Genera un backup completo listo para hosting. Permite seleccionar contenedor WordPress, contenedor DB y base de datos de forma interactiva, y arranca contenedores si estan apagados.

📁 Destino del backup: %USERPROFILE%\Desktop\wordpress-export\YYYY_MM_dd_HH-mm\

🔄 Restaurar URL local
1/4 Corregir URL antes de exportar
Usa wp search-replace para sustituir la URL local indicada (editable en tiempo de ejecucion, por defecto http://localhost:8181) por el dominio real de produccion.
2/4 Exportar base de datos
Ejecuta mysqldump dentro del contenedor de base de datos seleccionado y copia el resultado como export.sql en la carpeta de backup.
3/4 Exportar wp-content
Genera un .tar del directorio wp-content dentro del contenedor WordPress siguiendo symlinks (tar ch), lo copia al host y lo extrae como carpeta. El archivo wp-content.tar y la carpeta wp-content/ quedan ambos disponibles.
4/4 Restaurar URL local
Revierte el search-replace para dejar el entorno local tal y como estaba antes de la exportacion, incluso si hubo error en pasos intermedios.
Archivos generados en la carpeta de backup
wordpress-export/
    └── 2026_03_13_08-53/
    ├── export.sql        ← Importar en phpMyAdmin del hosting
    ├── wp-content/       ← Subir por FTP a public_html/wp-content
    └── wp-content.tar    ← Alternativa comprimida
⚠️ Importante: Al importar en el hosting, no tocar wp-config.php. Solo se importa el SQL y se sube el wp-content por FTP.
04

Importar backup al entorno local

📥
📄 importar-wordpress.bat

Restaura un backup WordPress en Docker local con deteccion interactiva de contenedores, URL local y credenciales de base de datos, ademas de navegador de carpetas.

📂 Ubicación por defecto: Busca backups en la carpeta padre donde se encuentra el propio .bat. Puedes navegar hacia arriba o hacia subcarpetas.
Paso previo Navegador de carpetas interactivo
Muestra un listado de carpetas y archivos numerados. Indica automaticamente que tipo de contenido importable hay en cada carpeta ([wp-content], [tar], [zip], [sql]). Puedes navegar hacia subcarpetas o volver atras con [0].
1/4 Copiar wp-content al contenedor
Acepta tres formatos de origen:
• TAR — copia y descomprime con tar xf
• ZIP — copia y descomprime con unzip
• Carpeta — copia directamente con docker cp

Tras descomprimir, hace cp -rf al directorio de WordPress y ajusta permisos con chown 1001:1001.
2/4 Importar base de datos
Copia el .sql seleccionado al contenedor MariaDB y lo importa en la base de datos que hayas elegido en el asistente.
3/4 Corregir URL
Ejecuta wp search-replace en WordPress usando la URL local detectada automaticamente (http://localhost:PUERTO) y el dominio real del backup que introduces en el asistente.
4/4 Reiniciar WordPress
Reinicia el contenedor WordPress seleccionado y espera unos segundos para que el sitio recargue con el nuevo contenido importado.
✅ Resultado: El backup queda disponible en http://localhost:PUERTO_DETECTADO; el script intenta autodetectar credenciales DB y permite elegir la base de datos antes de importar.
Formatos de wp-content soportados
Formato TAR (recomendado):
  wp-content.tar  ← generado por el script de exportación

Formato ZIP:
  wp-content.zip  ← compatible con backups externos

Carpeta directa:
  wp-content/     ← carpeta descomprimida manualmente
05

Menu principal de utilidades

🧭
📄 menu-utilidades.bat

Punto de entrada unico para trabajar con todos los scripts. Lanza cada utilidad en una ventana hija, espera a que termine y vuelve automaticamente al menu.

1/3 Seleccionar script por numero
Desde una sola pantalla puedes lanzar docker_mariadb_wordpress.bat, importar-wordpress.bat, exportar-wordpress-v3.bat y gestor_contenedores.bat, o abrir la documentación HTML en el navegador.
2/3 Retorno automatico al menu
Cada utilidad se ejecuta con espera (start /wait), por lo que al cerrar el script hijo reaparece el menu principal sin abrir otra consola manualmente.
3/3 Cerrar CMD desde el menu
Incluye una opcion dedicada [0] para salir y cerrar la ventana de comandos de forma limpia.
Opciones disponibles en el menu
1  ← docker_mariadb_wordpress.bat
2  ← importar-wordpress.bat
3  ← exportar-wordpress-v3.bat
4  ← gestor_contenedores.bat
5  ← abrir documentación
0  ← Cerrar CMD
⚠️ Nota: El menu no reemplaza a las utilidades: solo actua como lanzador central para simplificar el flujo de trabajo.
